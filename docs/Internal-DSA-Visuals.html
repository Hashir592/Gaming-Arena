1. Hash Table O(1) Access

Implementation: Separate Chaining using Linked Lists. An array of buckets stores pointers to list nodes.
When ID % Size results in the same index (e.g., Index 5), nodes are chained to handle the collision.

2. AVL Tree O(log n) Search

A Self-Balancing Binary Search Tree. Every node holds a Player Profile, sorted by ELO.
The "Balance Factor" ensures the height difference (Left vs Right) never exceeds 1, guaranteeing fast lookups.

3. Queue FIFO Fairness

Implementation: Singly Linked List with Head and Tail pointers.
Allows O(1) Enqueue (at Tail) and O(1) Dequeue (at Head).
