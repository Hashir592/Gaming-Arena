Hash Table (Separate Chaining)

O(1) Average Access

Player profiles are stored in a hash table using their Player ID as the key. The hash function ID % TableSize determines the bucket index. When collisions occur (multiple IDs hash to the same index), we chain them using a Linked List.

Matchmaking Queue (FIFO)

O(1) Enqueue/Dequeue

Players waiting for a match are held in a First-In-First-Out (FIFO) Queue. This ensures fairness: the player who has waited the longest gets matched first. Implemented using a Linked List with HEAD and REAR pointers for constant-time operations.

AVL Tree (Self-Balancing BST)

O(log n) Search

Players are organized by their ELO rating in a self-balancing AVL Tree. This enables the findClosest(targetELO) operation to find the most evenly matched opponent in logarithmic time.
